Quilt Playground: expansion in the browser

The whole pipeline runs client-side, no server. Editing the .html.ts.quilt source and pressing Expand & run (⌘/Ctrl+↵) sends it through the Quilt parser + expander (compiled to WebAssembly, driven by a tiny WASI shim), producing plain TypeScript whose html↖…↗ quotes have become quilt-wasm builder calls. That program is then imported and run against the same runtime, and its HTML is rendered below. The arrow glyphs are hard to type, so use the buttons (or their Alt shortcuts).
