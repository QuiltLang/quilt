# CLI & Scripts

## `quilt` — the main CLI

**File:** `rust/quilt/src/bin.rs`

The `quilt` binary has two subcommands.

### `quilt expand <file.rs.quilt>`

Expands a `.quilt` file to the corresponding source file (strips `.quilt` suffix).

```sh
quilt expand path/to/foo.rs.quilt     # produces foo.rs
quilt expand path/to/foo.py.quilt     # produces foo.py
quilt expand path/to/foo.wgsl.rs.quilt  # produces foo.wgsl.rs
```

Options:
- `-m, --multi <omni|bootstrap>` — select which `Multi` to use. Default `omni`. Use `bootstrap` only when expanding `mk_meta.rs.quilt` to regenerate `meta.rs`.

The generated file starts with a comment `//! DO NOT EDIT. GENERATED BY quilt expand ...`.

### `quilt run <file.rs.quilt> [args…]`

Expand a `.quilt` file to a temp file and run it immediately. The file's inner extension determines the runner:

| Extension | Runner                                                      |
|-----------|-------------------------------------------------------------|
| `.rs`     | `rust-script`                                               |
| `.py`     | `python3` (with `PYTHONPATH` set to include `quilt_python`) |

The runner is read from `Language::hashbang()`. If the language does not return a hashbang, `quilt run` errors.

Trailing arguments after the filename are forwarded to the script.

For Rust scripts, `quilt run` injects a `//! [dependencies]` cargo manifest block in the generated file pointing at the local quilt crate with the correct feature set.

```sh
quilt run examples/hello.rs.quilt World
quilt run examples/hello.py.quilt
quilt run examples/countdown.rs.quilt 5
```

### Shebang scripts

`.quilt` files can be used directly as executables with:

```sh
#!/usr/bin/env quilt run
```

as their first line. `quilt run` strips the shebang before expansion so the language parser doesn't see `#!`.

## `bin/` scripts

All scripts in `bin/` expect the `direnv` environment to be active (`direnv allow` from the repo root). They work from any directory when the env is active.

### `quilt`

```sh
quilt expand path/to/file.rs.quilt
quilt run path/to/file.rs.quilt [args]
```

A thin wrapper around `cargo run -p quilt --`. Passes all arguments to the `quilt` binary.

### `bootstrap`

Runs the full two-stage bootstrap: expands `mk_meta.rs.quilt` (stage 0) then runs the result to regenerate `meta.rs` (stage 1).

```sh
bootstrap
```

Equivalent to running `bootstrap0` then `bootstrap1` in order.

### `bootstrap0`

Stage 0 only: expand `mk_meta.rs.quilt` using `BootstrapMetaLanguage`.

```sh
bootstrap0
```

### `bootstrap1`

Stage 1 only: run the already-expanded `mk_meta.rs` with `rust-script` to regenerate `meta.rs`.

```sh
bootstrap1
```

### `build-py`

Build the `quilt_python` PyO3 extension module. Required before running `.py.quilt` files. Rebuild after editing the Python bindings.

```sh
build-py
```

Produces a `.so` / `.dylib` file in the `quilt_python/quilt/` directory that Python can import as `import quilt`.

### `ts-gen`

Regenerate the tree-sitter parser after editing `tree-sitter-quilt/grammar.js`.

```sh
ts-gen
```

## Cargo workspace commands

Run from `rust/` (the Cargo workspace root):

```sh
cargo build                       # build all workspace members
cargo test                        # run all tests
cargo test -p quilt node          # run tests matching "node" in the quilt crate
cargo clippy                      # lint
cargo fmt                         # format

cargo build -p quilt-lsp          # build the LSP server
cargo test -p quilt-lsp           # run LSP tests
```

## Environment variables

| Variable                  | Used by              | Description                                           |
|---------------------------|----------------------|-------------------------------------------------------|
| `RUST_LOG`                | `quilt`, `quilt-lsp` | `tracing` log filter (e.g. `debug`, `info`)           |
| `QUILT_LSP_RUST_ANALYZER` | `quilt-lsp`          | Override rust-analyzer command (whitespace-separated) |
| `PYTHONPATH`              | `quilt run` (Python) | Extended to include `quilt_python/` directory         |
